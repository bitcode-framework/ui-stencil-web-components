# Phase 4.5k: WASM + Native Plugins

**Date**: 15 July 2026
**Status**: Draft
**Depends On**: Phase 4.5j (Script Plugins + Runtime Extraction)
**Effort**: 2-3 weeks
**Scope**: `packages/go-json/runtime/`, `packages/go-json/lang/`, `packages/go-json-runtimes/wasm/`, `packages/go-json-runtimes/native/`

---

## 1. Overview

This phase adds two new plugin types to go-json:

1. **WASM Plugins** (`wasm:` import) — WebAssembly modules executed via [wazero](https://wazero.io/) (pure Go, zero CGO). Portable, sandboxed, high-performance.
2. **Native Plugins** (`plugin:` import) — Shared libraries (.so/.dll/.dylib) loaded via Go's `plugin` package or dlopen. Maximum performance, platform-specific.

Both plugin types follow the same `ScriptRuntime` interface defined in Phase 4.5j, enabling uniform usage from go-json programs.

---

## 2. Goals & Non-Goals

### Goals

- Add `wasm:` import type to go-json parser and VM
- Add `plugin:` import type to go-json parser and VM
- Implement WASM runtime in `packages/go-json-runtimes/wasm/` using wazero
- Implement native plugin runtime in `packages/go-json-runtimes/native/`
- Define WASM plugin protocol (function exports, memory management, JSON serialization)
- Define native plugin protocol (symbol lookup, calling convention)
- Support codegen for WASM and native plugin calls
- Maintain go-json core zero-dependency constraint

### Non-Goals

- WASM component model (too early, spec not stable)
- WASI preview 2 (use preview 1 for now)
- Cross-compilation of WASM modules (out of scope — users bring their own .wasm)
- Hot-reloading of native plugins
- Windows native plugins via Go's `plugin` package (not supported by Go)

## 4. WASM Plugin Design

### 4.1 Why wazero

| Criterion | wazero | wasmer-go | wasmtime-go |
|-----------|--------|-----------|-------------|
| Pure Go (no CGO) | ✅ | ❌ | ❌ |
| Zero external deps | ✅ | ❌ | ❌ |
| WASI support | ✅ (preview 1) | ✅ | ✅ |
| Performance | Good (JIT on supported platforms) | Better | Better |
| Cross-compile | ✅ (any GOOS/GOARCH) | ❌ | ❌ |
| Maintenance | Active | Declining | Active |

wazero is the only option that maintains go-json-runtimes' ability to cross-compile without CGO. This aligns with the project's "zero external dependency for core" philosophy.

### 4.2 WASM Plugin Protocol

WASM modules communicate with the host via a JSON-over-linear-memory protocol:

```
┌─────────────────────────────────────────────────────────────────┐
│ WASM Plugin Protocol                                             │
│                                                                  │
│ Host → WASM:                                                     │
│   1. Serialize args to JSON bytes                                │
│   2. Allocate memory in WASM (call wasm.malloc)                  │
│   3. Write JSON bytes to allocated memory                        │
│   4. Call exported function with (ptr, len) arguments            │
│                                                                  │
│ WASM → Host:                                                     │
│   1. Function processes input                                    │
│   2. Serialize result to JSON bytes                              │
│   3. Allocate memory for result (call wasm.malloc)               │
│   4. Write result JSON to memory                                 │
│   5. Return (ptr, len) as i64 (packed: high 32 = ptr, low 32 = len) │
│                                                                  │
│ Host reads result:                                               │
│   1. Unpack (ptr, len) from return value                         │
│   2. Read JSON bytes from WASM memory                            │
│   3. Deserialize to Go value                                     │
│   4. Free WASM memory (call wasm.free)                           │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 4.3 WASM Module Requirements

A valid go-json WASM plugin must export:

| Export | Signature | Purpose |
|--------|-----------|---------|
| `malloc` | `(size: i32) → ptr: i32` | Allocate memory for host→wasm data |
| `free` | `(ptr: i32, size: i32)` | Free previously allocated memory |
| `<function_name>` | `(ptr: i32, len: i32) → i64` | Business logic function |

The return value is a packed i64: `(result_ptr << 32) | result_len`.

### 4.4 WASM Host Functions (Bridge)

The host exposes bridge functions to WASM via imports:

```go
// packages/go-json-runtimes/wasm/host_functions.go

// Host functions available to WASM modules under "env" namespace:
//
// env.bridge_call(fn_ptr, fn_len, args_ptr, args_len) → i64
//   Calls a bridge function by name with JSON-serialized arguments.
//   Returns packed (ptr, len) pointing to JSON result in WASM memory.
//
// env.log(level, msg_ptr, msg_len)
//   Logs a message from WASM to the host logger.

func (w *WasmRuntime) defineHostFunctions(mod wazero.ModuleBuilder, bridge map[string]any) {
    mod.NewFunctionBuilder().
        WithFunc(func(ctx context.Context, m api.Module, fnPtr, fnLen, argsPtr, argsLen uint32) uint64 {
            // 1. Read function name from WASM memory
            fnName := readString(m, fnPtr, fnLen)
            // 2. Read args JSON from WASM memory
            argsJSON := readBytes(m, argsPtr, argsLen)
            // 3. Resolve and call bridge function
            result, err := callBridgeFunction(bridge, fnName, argsJSON)
            // 4. Write result back to WASM memory
            return writeResult(m, result, err)
        }).
        Export("bridge_call")
}
```

### 4.5 WASM Module Lifecycle

```go
// packages/go-json-runtimes/wasm/runtime.go

type WasmRuntime struct {
    engine  wazero.Runtime
    cache   map[string]wazero.CompiledModule // compile once, instantiate many
    config  WasmConfig
    mu      sync.RWMutex
}

type WasmConfig struct {
    Enabled       bool
    MaxMemoryMB   int           // per-instance memory limit (default: 64)
    MaxExecTime   time.Duration // per-call timeout (default: 30s)
    CompileCache  bool          // cache compiled modules (default: true)
    WASIEnabled   bool          // enable WASI (filesystem, env, etc.) (default: false)
}

func NewWasmRuntime(config WasmConfig) (*WasmRuntime, error) {
    ctx := context.Background()
    
    runtimeConfig := wazero.NewRuntimeConfig()
    if config.MaxMemoryMB > 0 {
        runtimeConfig = runtimeConfig.WithMemoryLimitPages(uint32(config.MaxMemoryMB * 16)) // 1 page = 64KB
    }
    
    engine := wazero.NewRuntimeWithConfig(ctx, runtimeConfig)
    
    return &WasmRuntime{
        engine: engine,
        cache:  make(map[string]wazero.CompiledModule),
        config: config,
    }, nil
}

// ScriptRuntime interface implementation:

func (w *WasmRuntime) Name() string { return "wasm" }

func (w *WasmRuntime) Extensions() []string { return []string{".wasm"} }

func (w *WasmRuntime) CanHandle(ext string) bool { return ext == ".wasm" }

func (w *WasmRuntime) Execute(ctx context.Context, script, function string, params, bridge map[string]any) (any, error) {
    // 1. Compile module (cached)
    compiled, err := w.getOrCompile(ctx, script)
    if err != nil {
        return nil, fmt.Errorf("wasm compile error: %w", err)
    }
    
    // 2. Instantiate with host functions
    instance, err := w.instantiate(ctx, compiled, bridge)
    if err != nil {
        return nil, fmt.Errorf("wasm instantiate error: %w", err)
    }
    defer instance.Close(ctx)
    
    // 3. Serialize params to JSON
    argsJSON, err := json.Marshal(params)
    if err != nil {
        return nil, fmt.Errorf("wasm args serialization error: %w", err)
    }
    
    // 4. Write args to WASM memory
    ptr, err := w.wasmMalloc(ctx, instance, uint32(len(argsJSON)))
    if err != nil {
        return nil, fmt.Errorf("wasm malloc error: %w", err)
    }
    if !instance.Memory().Write(ptr, argsJSON) {
        return nil, fmt.Errorf("wasm memory write failed")
    }
    
    // 5. Call function with timeout
    fn := instance.ExportedFunction(function)
    if fn == nil {
        return nil, fmt.Errorf("wasm function '%s' not exported", function)
    }
    
    timeoutCtx, cancel := context.WithTimeout(ctx, w.config.MaxExecTime)
    defer cancel()
    
    results, err := fn.Call(timeoutCtx, uint64(ptr), uint64(len(argsJSON)))
    if err != nil {
        return nil, fmt.Errorf("wasm execution error: %w", err)
    }
    
    // 6. Read result from WASM memory
    resultPacked := results[0]
    resultPtr := uint32(resultPacked >> 32)
    resultLen := uint32(resultPacked & 0xFFFFFFFF)
    
    resultJSON, ok := instance.Memory().Read(resultPtr, resultLen)
    if !ok {
        return nil, fmt.Errorf("wasm result read failed")
    }
    
    // 7. Free WASM memory
    w.wasmFree(ctx, instance, resultPtr, resultLen)
    
    // 8. Deserialize result
    var result any
    if err := json.Unmarshal(resultJSON, &result); err != nil {
        return nil, fmt.Errorf("wasm result deserialization error: %w", err)
    }
    
    return result, nil
}

func (w *WasmRuntime) Validate() error { return nil } // wazero is embedded, always available

func (w *WasmRuntime) Close() error {
    return w.engine.Close(context.Background())
}
```

### 4.6 WASM Plugin Authoring (User Guide)

Users can write WASM plugins in any language that compiles to WASM:

**Rust** (recommended — best WASM support):
```rust
// plugins/imgproc/src/lib.rs
use serde::{Deserialize, Serialize};

#[derive(Deserialize)]
struct ResizeArgs {
    args: Vec<serde_json::Value>,
}

#[derive(Serialize)]
struct ResizeResult {
    width: u32,
    height: u32,
    data: Vec<u8>,
}

#[no_mangle]
pub extern "C" fn resize(ptr: *const u8, len: usize) -> u64 {
    let input = unsafe { std::slice::from_raw_parts(ptr, len) };
    let args: ResizeArgs = serde_json::from_slice(input).unwrap();
    
    // ... resize logic ...
    
    let result = ResizeResult { width: 256, height: 256, data: vec![] };
    let json = serde_json::to_vec(&result).unwrap();
    
    let result_ptr = json.as_ptr() as u32;
    let result_len = json.len() as u32;
    std::mem::forget(json); // prevent deallocation
    
    ((result_ptr as u64) << 32) | (result_len as u64)
}

#[no_mangle]
pub extern "C" fn malloc(size: usize) -> *mut u8 {
    let mut buf = Vec::with_capacity(size);
    let ptr = buf.as_mut_ptr();
    std::mem::forget(buf);
    ptr
}

#[no_mangle]
pub extern "C" fn free(ptr: *mut u8, size: usize) {
    unsafe { Vec::from_raw_parts(ptr, 0, size); }
}
```

Compile: `cargo build --target wasm32-wasi --release`

### 4.7 WASM Instance Pooling

For high-throughput scenarios, WASM instances can be pooled. WASM pooling is DIFFERENT from process pooling (Node.js/Python) because WASM instances are in-process (no child process):

```go
type WasmInstancePool struct {
    compiled wazero.CompiledModule
    pool     chan api.Module
    config   WasmPoolConfig
}

type WasmPoolConfig struct {
    Size            int // number of pre-instantiated modules
    MaxMemoryMB     int // per-instance memory limit (maps to bitcode.yaml max_memory_mb)
    HardMaxMemoryMB int // absolute ceiling (maps to bitcode.yaml hard_max_memory_mb)
}

// Pre-instantiate N modules for reuse:
func NewWasmInstancePool(ctx context.Context, rt wazero.Runtime, compiled wazero.CompiledModule, config WasmPoolConfig) *WasmInstancePool {
    pool := make(chan api.Module, config.Size)
    for i := 0; i < config.Size; i++ {
        instance, _ := rt.InstantiateModule(ctx, compiled, wazero.NewModuleConfig())
        pool <- instance
    }
    return &WasmInstancePool{compiled: compiled, pool: pool, config: config}
}
```

### 4.8 WASM and BitCode Pool Config Mapping

WASM is an **embedded** runtime (like goja/yaegi) — it runs in-process, not as a child process. Therefore:

- **Process pool** (Size, MaxExecutions, CrashRecovery) → NOT applicable to WASM. WASM uses instance pooling instead.
- **Memory limits** (MaxMemoryMB, HardMaxMemoryMB) → APPLICABLE. Enforced via wazero's `WithMemoryLimitPages()`.
- **Timeout** (DefaultStepTimeout, MaxStepTimeout) → Enforced by CALLER via `context.Context`, same as all runtimes.

```yaml
# bitcode.yaml — WASM uses the same unified pool timeout config:
runtime:
  worker:
    # These apply to WASM calls routed to "worker" pool:
    default_step_timeout: "30s"    # → context.WithTimeout on WASM Execute()
    max_step_timeout: "5m"         # → ceiling enforced by BitCode executor
    max_memory_mb: 64              # → wazero WithMemoryLimitPages()
    hard_max_memory_mb: 256        # → absolute ceiling, NOT overridable

  wasm:
    enabled: true
    max_memory_mb: 64              # default per-instance (overridable by process)
    max_exec_time: "30s"           # fallback if no context deadline (standalone mode)
    compile_cache: true
    instance_pool_size: 4          # pre-instantiated instances per module
    wasi_enabled: false
```

**Timeout enforcement for WASM** follows the same flow as all runtimes:
```
BitCode Executor                    go-json-runtimes/wasm
────────────────                    ─────────────────────
1. Resolve timeout from hierarchy   
   (step→process→module→project)    
2. ctx = context.WithTimeout(       
     parentCtx, resolvedTimeout)    
3. wasmRuntime.Execute(ctx, ...)  → 4. fn.Call(ctx, ...) — wazero respects ctx
                                       If ctx cancelled → execution interrupted
```

For **standalone go-json CLI** (no BitCode), WASM uses `WasmConfig.MaxExecTime` as fallback timeout when no context deadline is set.

## 5. Native Plugin Design

### 5.1 Platform Support

| Platform | Mechanism | Status |
|----------|-----------|--------|
| Linux | Go `plugin` package | ✅ Supported |
| macOS | Go `plugin` package | ✅ Supported |
| Windows | Not supported by Go `plugin` | ❌ Not supported |
| FreeBSD | Go `plugin` package | ⚠️ Experimental |

**Priority**: Low — native plugins are platform-specific and limit portability. WASM is the recommended alternative for performance-critical plugins.

### 5.2 Native Plugin Protocol

Native plugins are Go shared libraries that export functions matching a specific signature:

```go
// Plugin must export functions with this signature:
// func FunctionName(args map[string]any) (any, error)

// Example: plugins/fast.go
package main

import "fmt"

// Compute performs heavy computation.
func Compute(args map[string]any) (any, error) {
    data, ok := args["args"].([]any)
    if !ok {
        return nil, fmt.Errorf("invalid args")
    }
    // ... heavy computation ...
    return map[string]any{"result": 42}, nil
}

// Manifest returns the list of exported functions.
// This is REQUIRED for function discovery.
func Manifest() []string {
    return []string{"Compute", "Transform", "Validate"}
}
```

Build: `go build -buildmode=plugin -o fast.so fast.go`

### 5.3 Native Runtime Implementation

```go
// packages/go-json-runtimes/native/runtime.go

package native

import (
    "context"
    "fmt"
    "path/filepath"
    "plugin"
    "runtime"
    "sync"
)

type NativeRuntime struct {
    plugins map[string]*loadedPlugin
    config  NativeConfig
    mu      sync.RWMutex
}

type NativeConfig struct {
    Enabled     bool
    AllowedDirs []string // directories from which plugins can be loaded
}

type loadedPlugin struct {
    plugin    *plugin.Plugin
    functions map[string]plugin.Symbol
    manifest  []string
}

func NewNativeRuntime(config NativeConfig) (*NativeRuntime, error) {
    if runtime.GOOS == "windows" {
        return nil, fmt.Errorf("native plugins not supported on Windows")
    }
    return &NativeRuntime{
        plugins: make(map[string]*loadedPlugin),
        config:  config,
    }, nil
}

func (n *NativeRuntime) Name() string { return "native" }

func (n *NativeRuntime) Extensions() []string {
    switch runtime.GOOS {
    case "linux":
        return []string{".so"}
    case "darwin":
        return []string{".dylib", ".so"}
    default:
        return []string{".so"}
    }
}

func (n *NativeRuntime) CanHandle(ext string) bool {
    for _, e := range n.Extensions() {
        if ext == e {
            return true
        }
    }
    return false
}

func (n *NativeRuntime) Execute(ctx context.Context, script, function string, params, bridge map[string]any) (any, error) {
    // 1. Load plugin (cached)
    loaded, err := n.getOrLoad(script)
    if err != nil {
        return nil, fmt.Errorf("native plugin load error: %w", err)
    }
    
    // 2. Lookup function symbol
    sym, ok := loaded.functions[function]
    if !ok {
        return nil, fmt.Errorf("native plugin function '%s' not found in %s", function, filepath.Base(script))
    }
    
    // 3. Assert function signature
    fn, ok := sym.(func(map[string]any) (any, error))
    if !ok {
        return nil, fmt.Errorf("native plugin function '%s' has wrong signature (expected func(map[string]any) (any, error))", function)
    }
    
    // 4. Call with context timeout
    type result struct {
        value any
        err   error
    }
    done := make(chan result, 1)
    go func() {
        v, e := fn(params)
        done <- result{v, e}
    }()
    
    select {
    case r := <-done:
        return r.value, r.err
    case <-ctx.Done():
        return nil, ctx.Err()
    }
}

func (n *NativeRuntime) Validate() error {
    if runtime.GOOS == "windows" {
        return fmt.Errorf("native plugins not supported on Windows (use wasm: instead)")
    }
    return nil
}

func (n *NativeRuntime) Close() error { return nil }

func (n *NativeRuntime) getOrLoad(path string) (*loadedPlugin, error) {
    n.mu.RLock()
    if loaded, ok := n.plugins[path]; ok {
        n.mu.RUnlock()
        return loaded, nil
    }
    n.mu.RUnlock()
    
    // Validate path is in allowed directories
    if err := n.validatePath(path); err != nil {
        return nil, err
    }
    
    // Load plugin
    p, err := plugin.Open(path)
    if err != nil {
        return nil, fmt.Errorf("failed to open plugin: %w", err)
    }
    
    // Get manifest
    manifestSym, err := p.Lookup("Manifest")
    if err != nil {
        return nil, fmt.Errorf("plugin missing Manifest() function: %w", err)
    }
    manifestFn, ok := manifestSym.(func() []string)
    if !ok {
        return nil, fmt.Errorf("plugin Manifest has wrong signature")
    }
    manifest := manifestFn()
    
    // Load all exported functions
    functions := make(map[string]plugin.Symbol)
    for _, name := range manifest {
        sym, err := p.Lookup(name)
        if err != nil {
            return nil, fmt.Errorf("plugin declares '%s' in manifest but symbol not found: %w", name, err)
        }
        functions[name] = sym
    }
    
    loaded := &loadedPlugin{
        plugin:    p,
        functions: functions,
        manifest:  manifest,
    }
    
    n.mu.Lock()
    n.plugins[path] = loaded
    n.mu.Unlock()
    
    return loaded, nil
}

func (n *NativeRuntime) validatePath(path string) error {
    abs, err := filepath.Abs(path)
    if err != nil {
        return err
    }
    for _, dir := range n.config.AllowedDirs {
        allowedAbs, _ := filepath.Abs(dir)
        if filepath.HasPrefix(abs, allowedAbs) {
            return nil
        }
    }
    return fmt.Errorf("plugin path '%s' not in allowed directories", path)
}
```

## 7. Codegen Strategy

### 7.1 WASM Plugin Calls

WASM is portable — the same `.wasm` file works in all codegen targets:

| Target | Generated Code |
|--------|---------------|
| Go | Load .wasm with wazero, call function (same as runtime) |
| JavaScript | Load .wasm with WebAssembly API (browser/Node.js native) |
| Python | Load .wasm with wasmer-python or wasmtime-python |

```go
// Go codegen for: {"import": {"img": "wasm:./plugins/imgproc.wasm"}}

// Generated:
import "github.com/tetratelabs/wazero"

var imgModule wazero.CompiledModule

func init() {
    // Compile WASM module at startup
    wasmBytes, _ := os.ReadFile("./plugins/imgproc.wasm")
    imgModule, _ = wazeroRuntime.CompileModule(ctx, wasmBytes)
}

func img_resize(data any, width, height int) (any, error) {
    instance, _ := wazeroRuntime.InstantiateModule(ctx, imgModule, moduleConfig)
    defer instance.Close(ctx)
    // ... marshal args, call, unmarshal result ...
}
```

```javascript
// JavaScript codegen for: {"import": {"img": "wasm:./plugins/imgproc.wasm"}}

// Generated:
const wasmBytes = await fetch('./plugins/imgproc.wasm').then(r => r.arrayBuffer());
const { instance } = await WebAssembly.instantiate(wasmBytes, imports);

function img_resize(data, width, height) {
    // ... write args to memory, call, read result ...
    return instance.exports.resize(ptr, len);
}
```

### 7.2 Native Plugin Calls

Native plugins are NOT portable — codegen generates interface stubs:

| Target | Generated Code |
|--------|---------------|
| Go | Direct plugin.Open + symbol lookup (same platform only) |
| JavaScript | Interface stub with error: "native plugins not supported in JS target" |
| Python | Interface stub with ctypes/cffi fallback (if .so is compatible) |

```go
// Go codegen for: {"import": {"fast": "plugin:./plugins/fast.so"}}

// Generated:
import "plugin"

var fastPlugin *plugin.Plugin
var fast_Compute func(map[string]any) (any, error)

func init() {
    p, err := plugin.Open("./plugins/fast.so")
    if err != nil {
        panic("native plugin not available: " + err.Error())
    }
    fastPlugin = p
    sym, _ := p.Lookup("Compute")
    fast_Compute = sym.(func(map[string]any) (any, error))
}
```

## 8. Security Considerations

### 8.1 WASM Sandbox

WASM is inherently sandboxed:
- No filesystem access (unless WASI enabled)
- No network access (unless host functions provide it)
- Memory isolated (linear memory, bounds-checked)
- Cannot escape sandbox

**WASI considerations**: If `WASIEnabled: true`, the WASM module gets limited filesystem access. This should be restricted to specific directories:

```go
type WasmConfig struct {
    // ...
    WASIEnabled   bool
    WASIDirs      map[string]string // guest path → host path mapping
    WASIEnv       map[string]string // environment variables visible to WASM
}
```

### 8.2 Native Plugin Security

Native plugins are **NOT sandboxed** — they run in the same process with full access. Security measures:

1. **AllowedDirs**: Only load plugins from whitelisted directories
2. **Path validation**: Prevent path traversal
3. **No hot-reload**: Once loaded, a plugin cannot be replaced (prevents TOCTOU attacks)
4. **Signature verification** (future): Verify plugin hash/signature before loading

### 8.3 Bridge Exposure

Both WASM and native plugins receive the same `map[string]any` bridge as script plugins. The bridge is built by the host (BitCode) with security rules already applied. Plugins cannot bypass security rules.

## 12. Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| wazero performance insufficient | Slow WASM execution | wazero has JIT on amd64/arm64; for interpreted platforms, document performance expectations |
| WASM memory protocol complex | Bugs in serialization | Extensive tests with various data types; consider extism protocol as alternative |
| Native plugins crash host process | Entire server crashes | Document risk clearly; recommend WASM over native; consider process isolation (future) |
| Test WASM modules hard to build | CI complexity | Pre-compile test .wasm files, commit to repo; Makefile for rebuilding |
| Go plugin package limitations | Can't unload plugins, Windows unsupported | Document limitations; recommend WASM as primary; native is "escape hatch" |

## 13. Future Considerations

### 13.1 Extism Protocol (Alternative to Custom Protocol)

[Extism](https://extism.org/) provides a standard WASM plugin protocol with SDKs for many languages. If our custom protocol proves too complex, we could adopt Extism's protocol instead. This would give users access to existing Extism PDKs (Plugin Development Kits) for Rust, Go, C, AssemblyScript, etc.

### 13.2 WASM Component Model

When the WASM Component Model stabilizes, it will provide:
- Typed interfaces (no JSON serialization needed)
- Composable components
- Standard resource types

This is a future upgrade path that doesn't affect Phase 4.5k's design.

### 13.3 Process Isolation for Native Plugins

For production safety, native plugins could run in a separate process with IPC. This would prevent plugin crashes from taking down the host. This is out of scope for Phase 4.5k but noted as a future enhancement.
