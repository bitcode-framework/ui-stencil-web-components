## 2. Features

### 2.1 Lambda Expressions: `fn(x) => expr`

#### 2.1.1 Syntax

```
fn(params) => body           // anonymous lambda (not recursive)
fn name(params) => body      // named lambda (CAN recurse)
```

Where:
- `name` = optional identifier (enables self-reference for recursion)
- `params` = comma-separated identifiers (zero or more)
- `body` = any valid expr-lang expression (including `let` chains)

```jsonc
// Anonymous lambda — NOT recursive
{"let": "double", "expr": "fn(x) => x * 2"}
{"let": "add", "expr": "fn(a, b) => a + b"}
{"let": "greet", "expr": "fn() => 'Hello, World!'"}
{"let": "classify", "expr": "fn(age) => age >= 18 ? 'adult' : 'minor'"}

// Named lambda — CAN recurse
{"let": "factorial", "expr": "fn factorial(n) => n <= 1 ? 1 : n * factorial(n - 1)"}
{"let": "fib", "expr": "fn fib(n) => n <= 1 ? n : fib(n-1) + fib(n-2)"}

// Multi-statement (expr-lang let chains)
{"let": "transform", "expr": "fn(x) => let y = x * 2; let z = y + 10; z * z"}

// Using stdlib functions
{"let": "normalize", "expr": "fn(s) => lower(trim(s))"}

// Composition — lambdas calling other lambdas
{"let": "addTax", "expr": "fn(price) => price * 1.11"}
{"let": "discount", "expr": "fn(price) => price * 0.8"}
{"let": "finalPrice", "expr": "fn(price) => discount(addTax(price))"}

// Named recursive lambda accessing other variables + lambdas
{"let": "multiplier", "value": 3},
{"let": "transform", "expr": "fn(x) => x * multiplier"},
{"let": "treeSum", "expr": "fn treeSum(node) => isNil(node) ? 0 : transform(node.value) + treeSum(node.left) + treeSum(node.right)"}
```

#### 2.1.2 Usage

```jsonc
// Direct call
{"let": "result", "expr": "double(5)"}                    // 10

// Pass to higher-order functions
{"let": "doubled", "expr": "mapFn([1,2,3], double)"}      // [2,4,6]

// Inline lambda
{"let": "evens", "expr": "filterFn([1,2,3,4], fn(x) => x % 2 == 0)"}  // [2,4]

// Composition
{"let": "total", "expr": "finalPrice(100)"}                // 88.8
```

#### 2.1.3 Scope Capture & Recursion

**Snapshot capture:** Lambdas capture variables at definition time (snapshot, not reference):

```jsonc
{"let": "multiplier", "value": 3},
{"let": "multiply", "expr": "fn(x) => x * multiplier"},
{"set": "multiplier", "value": 10},
{"let": "result", "expr": "multiply(5)"}
// result = 15 (captured multiplier=3, not current 10)
```

**Rationale:** Snapshot capture is:
- Predictable (no spooky action at a distance)
- Codegen-friendly (Go closures capture by reference — snapshot avoids this mismatch)
- Safe (no dangling references to out-of-scope variables)

**Named lambda recursion:** Named lambdas (`fn name(x) => ...`) inject their own name into their eval environment, enabling self-reference:

```jsonc
{"let": "factorial", "expr": "fn factorial(n) => n <= 1 ? 1 : n * factorial(n - 1)"}
// factorial(5) → 5 * factorial(4) → ... → 120
```

**Recursion depth limit:** Named lambdas enforce `vm.limits.MaxDepth` which is resolved from the limits hierarchy:

```
HardLimits (10000)           ← absolute ceiling, hardcoded, never overridable
    ↑ cap
WithLimits (runtime option)  ← host/platform sets this (default = DefaultLimits = 1000)
    ↑ cap
Program "limits" JSON        ← user can only LOWER from runtime option, never raise
```

Default depth = 1000 (sane default protecting against accidental infinite recursion). Exceeding the limit produces a graceful error:

```
lambda 'factorial': recursion depth limit (1000) exceeded
```

**To increase depth limit:**
- Standalone CLI: `go-json run program.json --max-depth 5000` (internally sets `WithLimits`)
- Embedded: `runtime.WithLimits(runtime.Limits{MaxDepth: 5000})`
- Program JSON: can only LOWER (e.g., `"limits": {"max_depth": 500}`) — security by design

**What lambdas CAN access:**
- Variables defined before the lambda (snapshot)
- Other lambdas defined before (callable)
- Named functions from `functions` block
- Struct methods (via instance variable)
- Bridge functions (`bc.*`, `http.*`, `fs.*`, etc.)
- Stdlib functions (`upper`, `trim`, `crypto.*`, etc.)
- Implicit variables (`session.*`, `execution.*`, `input.*`)
- Constants and enums (if defined)
- Themselves (named lambdas only — for recursion)

**What lambdas CANNOT access:**
- Variables defined AFTER the lambda (snapshot semantics)
- Variables in deeper scopes (isolated)
- Mutate captured variables (snapshot, not reference)

**Step 2: Lambda Parser**

```go
// lang/lambda.go
package lang

import (
    "strings"
    "sync/atomic"
)

type Lambda struct {
    Name   string   // empty = anonymous, non-empty = named (can recurse)
    Params []string
    Body   string
}

// ParseLambda parses lambda expressions:
//   "fn(x, y) => x + y"              → anonymous lambda
//   "fn factorial(n) => n * f(n-1)"  → named lambda (recursive)
// Returns nil if not a lambda expression.
func ParseLambda(expr string) *Lambda {
    expr = strings.TrimSpace(expr)
    if !strings.HasPrefix(expr, "fn") {
        return nil
    }

    rest := strings.TrimSpace(expr[2:]) // skip "fn"
    if len(rest) == 0 {
        return nil
    }

    // Determine: anonymous fn(x) or named fn name(x)
    var name string
    if rest[0] != '(' {
        // Named lambda: "fn factorial(n) => ..."
        // Find where name ends (next '(' or space)
        idx := strings.IndexAny(rest, "( ")
        if idx == -1 {
            return nil
        }
        name = strings.TrimSpace(rest[:idx])
        rest = strings.TrimSpace(rest[idx:])
    }

    // Now rest starts with "("
    if len(rest) == 0 || rest[0] != '(' {
        return nil
    }

    // Find closing paren
    parenDepth := 0
    closeIdx := -1
    for i := 0; i < len(rest); i++ {
        if rest[i] == '(' {
            parenDepth++
        } else if rest[i] == ')' {
            parenDepth--
            if parenDepth == 0 {
                closeIdx = i
                break
            }
        }
    }
    if closeIdx == -1 {
        return nil
    }

    // Extract params
    paramStr := strings.TrimSpace(rest[1:closeIdx])
    var params []string
    if paramStr != "" {
        for _, p := range strings.Split(paramStr, ",") {
            params = append(params, strings.TrimSpace(p))
        }
    }

    // Find "=>"
    after := strings.TrimSpace(rest[closeIdx+1:])
    if !strings.HasPrefix(after, "=>") {
        return nil
    }

    body := strings.TrimSpace(after[2:])
    if body == "" {
        return nil
    }

    return &Lambda{Name: name, Params: params, Body: body}
}
```

**Step 3: Lambda Compilation to Go func**

```go
// lang/lambda.go (continued)

// CompileLambda creates a Go function from a Lambda definition.
// capturedEnv is a snapshot of the current scope at definition time.
// maxDepth limits recursion for named lambdas (from vm.limits.MaxDepth).
func CompileLambda(lambda *Lambda, engine ExprEngine, capturedEnv map[string]any, maxDepth int) func(...any) (any, error) {
    if maxDepth <= 0 {
        maxDepth = 1000 // safe default
    }

    var self func(...any) (any, error)
    var depth int64 // atomic counter for recursion depth

    self = func(args ...any) (any, error) {
        // Depth limit enforcement (named lambdas only)
        if lambda.Name != "" {
            current := atomic.AddInt64(&depth, 1)
            defer atomic.AddInt64(&depth, -1)
            if current > int64(maxDepth) {
                return nil, fmt.Errorf("lambda '%s': recursion depth limit (%d) exceeded",
                    lambda.Name, maxDepth)
            }
        }

        // Create evaluation environment: captured scope + param bindings
        evalEnv := make(map[string]any, len(capturedEnv)+len(lambda.Params)+1)
        for k, v := range capturedEnv {
            evalEnv[k] = v
        }

        // Bind positional args to param names
        for i, param := range lambda.Params {
            if i < len(args) {
                evalEnv[param] = args[i]
            } else {
                evalEnv[param] = nil
            }
        }

        // Self-reference for named lambdas (enables recursion)
        if lambda.Name != "" {
            evalEnv[lambda.Name] = self
        }

        // Evaluate body as expr-lang expression
        return engine.Eval(lambda.Body, evalEnv)
    }

    return self
}
```

**Step 4: Integration with Expression Evaluation**

```go
// In VM.evalExpr or ExprEngine.Eval:
func (vm *VM) evalExpr(expression string, stepIndex int) (any, error) {
    // Pre-process: find and compile lambdas in the expression
    env := vm.scope.ToMap()
    processedExpr, lambdaFuncs := preprocessLambdas(expression, vm.engine, env)

    // Inject compiled lambdas into environment
    for name, fn := range lambdaFuncs {
        env[name] = fn
    }

    // Evaluate with expr-lang
    return vm.engine.Eval(processedExpr, env)
}

// preprocessLambdas finds lambda expressions and replaces them with
// references to compiled Go functions.
func preprocessLambdas(expr string, engine ExprEngine, env map[string]any) (string, map[string]func(...any)(any, error)) {
    funcs := make(map[string]func(...any)(any, error))

    // Strategy: find top-level "fn(" patterns, compile each,
    // replace with a generated unique name like "__lambda_0"
    // This handles: fn(x) => x * 2 as standalone assignment
    // AND: mapFn(items, fn(x) => x * 2) as inline argument

    // For standalone assignment (let double = fn(x) => x * 2):
    // The entire expression IS the lambda — compile and return as func
    if lambda := ParseLambda(expr); lambda != nil {
        return "", map[string]func(...any)(any,error){
            "__direct_lambda": CompileLambda(lambda, engine, env),
        }
    }

    // For inline lambdas (mapFn(items, fn(x) => x * 2)):
    // Find lambda subexpressions, compile each, replace with placeholder
    processed := expr
    idx := 0
    for {
        pos := findLambdaStart(processed)
        if pos == -1 {
            break
        }
        end := findLambdaEnd(processed, pos)
        lambdaStr := processed[pos:end]
        lambda := ParseLambda(lambdaStr)
        if lambda == nil {
            break
        }
        name := fmt.Sprintf("__lambda_%d", idx)
        funcs[name] = CompileLambda(lambda, engine, env)
        processed = processed[:pos] + name + processed[end:]
        idx++
    }

    return processed, funcs
}
```

```go
// stdlib/higher_order.go
package stdlib

func RegisterHigherOrder(r *Registry) {
    r.Register(expr.Function("mapFn", func(params ...any) (any, error) {
        arr, ok := toAnySlice(params[0])
        if !ok {
            return nil, fmt.Errorf("mapFn: first argument must be an array")
        }
        fn, ok := params[1].(func(...any) (any, error))
        if !ok {
            return nil, fmt.Errorf("mapFn: second argument must be a function")
        }
        result := make([]any, len(arr))
        for i, item := range arr {
            val, err := fn(item)
            if err != nil {
                return nil, fmt.Errorf("mapFn: error at index %d: %w", i, err)
            }
            result[i] = val
        }
        return result, nil
    }))

    r.Register(expr.Function("filterFn", func(params ...any) (any, error) {
        arr, ok := toAnySlice(params[0])
        if !ok {
            return nil, fmt.Errorf("filterFn: first argument must be an array")
        }
        fn, ok := params[1].(func(...any) (any, error))
        if !ok {
            return nil, fmt.Errorf("filterFn: second argument must be a function")
        }
        var result []any
        for _, item := range arr {
            val, err := fn(item)
            if err != nil {
                return nil, err
            }
            if b, ok := val.(bool); ok && b {
                result = append(result, item)
            }
        }
        if result == nil {
            result = []any{}
        }
        return result, nil
    }))

    r.Register(expr.Function("reduceFn", func(params ...any) (any, error) {
        arr, ok := toAnySlice(params[0])
        if !ok {
            return nil, fmt.Errorf("reduceFn: first argument must be an array")
        }
        fn, ok := params[1].(func(...any) (any, error))
        if !ok {
            return nil, fmt.Errorf("reduceFn: second argument must be a function")
        }
        if len(arr) == 0 {
            if len(params) > 2 {
                return params[2], nil
            }
            return nil, nil
        }
        acc := arr[0]
        if len(params) > 2 {
            acc = params[2]
        }
        start := 0
        if len(params) <= 2 {
            start = 1
        }
        for i := start; i < len(arr); i++ {
            val, err := fn(acc, arr[i])
            if err != nil {
                return nil, err
            }
            acc = val
        }
        return acc, nil
    }))

    r.Register(expr.Function("applyEach", func(params ...any) (any, error) {
        arr, ok := toAnySlice(params[0])
        if !ok {
            return nil, fmt.Errorf("applyEach: first argument must be an array")
        }
        fn, ok := params[1].(func(...any) (any, error))
        if !ok {
            return nil, fmt.Errorf("applyEach: second argument must be a function")
        }
        for _, item := range arr {
            if _, err := fn(item); err != nil {
                return nil, err
            }
        }
        return nil, nil
    }))

    r.Register(expr.Function("sortFn", func(params ...any) (any, error) {
        arr, ok := toAnySlice(params[0])
        if !ok {
            return nil, fmt.Errorf("sortFn: first argument must be an array")
        }
        fn, ok := params[1].(func(...any) (any, error))
        if !ok {
            return nil, fmt.Errorf("sortFn: second argument must be a function")
        }
        result := make([]any, len(arr))
        copy(result, arr)
        sort.Slice(result, func(i, j int) bool {
            val, _ := fn(result[i], result[j])
            if b, ok := val.(bool); ok {
                return b
            }
            return false
        })
        return result, nil
    }))
}
```

#### 2.1.5 Codegen

| go-json | Go | JavaScript | Python |
|---------|-----|-----------|--------|
| `fn(x) => x * 2` | `func(x interface{}) interface{} { return x.(int) * 2 }` | `(x) => x * 2` | `lambda x: x * 2` |
| `fn(a, b) => a + b` | `func(a, b interface{}) interface{} { return a.(int) + b.(int) }` | `(a, b) => a + b` | `lambda a, b: a + b` |
| `fn(x) => let y = x*2; y+10` | `func(x interface{}) interface{} { y := x.(int)*2; return y+10 }` | `(x) => { const y = x*2; return y+10; }` | `def __fn(x):\n    y = x*2\n    return y+10` |
| `fn() => crypto.uuid()` | `func() interface{} { return cryptoUUID() }` | `() => crypto.uuid()` | `lambda: crypto_uuid()` |
| `fn factorial(n) => n<=1 ? 1 : n*factorial(n-1)` | `var factorial func(any) any; factorial = func(n any) any { if n.(int)<=1 { return 1 }; return n.(int)*factorial(n.(int)-1) }` | `const factorial = (n) => n<=1 ? 1 : n*factorial(n-1)` | `def factorial(n): return 1 if n<=1 else n*factorial(n-1)` |

**Python caveat:** Multi-statement lambdas (with `let`) must be generated as `def` functions since Python `lambda` only supports single expressions.

**Codegen implementation:**

```go
// codegen/lambda.go
func (g *GoGenerator) generateLambda(lambda *Lambda) string {
    params := strings.Join(lambda.Params, ", ")
    // Check if body has let statements
    if strings.Contains(lambda.Body, "let ") {
        // Multi-statement: generate full function
        return fmt.Sprintf("func(%s interface{}) interface{} {\n%s\n}", params, g.generateLetChain(lambda.Body))
    }
    return fmt.Sprintf("func(%s interface{}) interface{} { return %s }", params, g.translateExpr(lambda.Body))
}

func (g *JSGenerator) generateLambda(lambda *Lambda) string {
    params := strings.Join(lambda.Params, ", ")
    if strings.Contains(lambda.Body, "let ") {
        return fmt.Sprintf("(%s) => { %s }", params, g.generateLetChain(lambda.Body))
    }
    return fmt.Sprintf("(%s) => %s", params, g.translateExpr(lambda.Body))
}

func (g *PythonGenerator) generateLambda(lambda *Lambda) string {
    params := strings.Join(lambda.Params, ", ")
    if strings.Contains(lambda.Body, "let ") {
        // Must use def for multi-statement
        name := g.nextLambdaName()
        g.emitDef(name, params, lambda.Body)
        return name
    }
    return fmt.Sprintf("lambda %s: %s", params, g.translateExpr(lambda.Body))
}
```

```go
// In vm.go callFunctionUnified, add fallback after program.Functions lookup fails:
func (vm *VM) callFunctionUnified(name string, ...) (any, error) {
    fn, ok := vm.program.Functions[name]
    if !ok {
        // Fallback 1: check if name is a variable containing a function name (string)
        if val, _, found := vm.scope.Get(name); found {
            switch v := val.(type) {
            case string:
                // Variable contains function name — resolve
                fn, ok = vm.program.Functions[v]
                if ok {
                    // Continue with resolved function
                    break
                }
                // Also check scope for wrapped functions
                if scopeVal, _, found := vm.scope.Get(v); found {
                    if callable, isFunc := scopeVal.(func(...any) (any, error)); isFunc {
                        return vm.callNativeFunc(callable, positionalWith, literalArgs, stepIndex)
                    }
                }
                return nil, RuntimeError("FUNC_NOT_FOUND", fmt.Sprintf("function '%s' (resolved from variable '%s') not defined", v, name), stepIndex)

            case func(...any) (any, error):
                // Variable contains a callable (lambda) — call directly
                return vm.callNativeFunc(v, positionalWith, literalArgs, stepIndex)
            }
        }

        if !ok {
            // Original error handling...
        }
    }
    // ... rest of existing logic
}

func (vm *VM) callNativeFunc(fn func(...any)(any, error), withArgs []string, args []any, stepIndex int) (any, error) {
    var callArgs []any
    if args != nil {
        callArgs = args
    } else if withArgs != nil {
        callArgs = make([]any, len(withArgs))
        for i, expr := range withArgs {
            val, err := vm.evalExpr(expr, stepIndex)
            if err != nil {
                return nil, err
            }
            callArgs[i] = val
        }
    }
    return fn(callArgs...)
}
```

**VM Execution:**

```go
func (vm *VM) executeSleep(n *SleepNode) error {
    var ms int
    switch v := n.Duration.(type) {
    case int:
        ms = v
    case string:
        val, err := vm.evalExpr(v, n.StepIndex)
        if err != nil {
            return err
        }
        f, ok := toFloat64(val)
        if !ok {
            return RuntimeError("TYPE_ERROR", "sleep duration must be a number", n.StepIndex)
        }
        ms = int(f)
    }

    if ms <= 0 {
        return nil
    }
    if ms > 300000 { // 5 minute max
        return LimitError("SLEEP_LIMIT", "sleep duration cannot exceed 300000ms (5 minutes)", n.StepIndex)
    }

    select {
    case <-time.After(time.Duration(ms) * time.Millisecond):
        return nil
    case <-vm.ctx.Done():
        return vm.ctx.Err()
    }
}
```

### 2.4 `retry` Step

```jsonc
{
  "retry": {
    "steps": [
      {"let": "resp", "call": "http.get", "args": ["https://api.example.com/data"]}
    ],
    "max": 3,
    "delay": 1000,
    "backoff": "exponential"
  }
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `steps` | `[]step` | required | Steps to retry |
| `max` | `int` | `3` | Maximum attempts |
| `delay` | `int` | `1000` | Base delay in ms |
| `backoff` | `string` | `"fixed"` | `"fixed"`, `"linear"`, `"exponential"` |

**Backoff calculation:**
- `fixed`: delay ms every time
- `linear`: delay * attempt ms
- `exponential`: delay * 2^(attempt-1) ms

**AST Node:**

```go
type RetryNode struct {
    Steps     []Node
    Max       int
    Delay     int
    Backoff   string // "fixed", "linear", "exponential"
    StepIndex int
}
```

**VM Execution:**

```go
func (vm *VM) executeRetry(n *RetryNode) (any, error) {
    var lastErr error
    for attempt := 1; attempt <= n.Max; attempt++ {
        result, err := vm.executeSteps(n.Steps)
        if err == nil {
            return result, nil
        }
        lastErr = err

        if attempt < n.Max {
            delay := calculateBackoff(n.Delay, attempt, n.Backoff)
            select {
            case <-time.After(time.Duration(delay) * time.Millisecond):
            case <-vm.ctx.Done():
                return nil, vm.ctx.Err()
            }
        }
    }
    return nil, fmt.Errorf("retry exhausted after %d attempts: %w", n.Max, lastErr)
}

func calculateBackoff(baseDelay, attempt int, strategy string) int {
    switch strategy {
    case "linear":
        return baseDelay * attempt
    case "exponential":
        return baseDelay * (1 << (attempt - 1)) // 2^(attempt-1)
    default: // "fixed"
        return baseDelay
    }
}
```

```go
    }

        msg := fmt.Sprintf("assertion failed: %s", n.Condition)
```

```go
}

// In compile-time validation of "set" steps:
if compiler.protectedNames[setTarget] {
    return CompileError("CONST_REASSIGN",
        fmt.Sprintf("cannot reassign constant '%s'", setTarget), stepIndex)
}
```

**VM:** Constants injected into root scope before execution:

```go
// In VM.Execute, after scope creation:
for name, value := range vm.program.Constants {
    vm.scope.Declare(name, value, "const")
}
```

### 2.7 Enum System

```jsonc
{
  "enums": {
    "Status": ["draft", "confirmed", "done", "cancelled"],
    "Priority": {"LOW": 1, "MEDIUM": 2, "HIGH": 3, "CRITICAL": 4}
  }
}
```

**Access:**

```jsonc
{"let": "s", "expr": "Status.draft"}           // "draft"
{"let": "p", "expr": "Priority.HIGH"}          // 3
{"if": "order.status == Status.confirmed", "then": [...]}
```

**Implementation:** Enums are compiled to `map[string]any` and injected into scope:

```go
// Array enum: ["draft", "confirmed"] → {"draft": "draft", "confirmed": "confirmed"}
// Map enum: {"LOW": 1, "HIGH": 3} → as-is

for name, def := range program.Enums {
    switch v := def.(type) {
    case []any:
        enumMap := make(map[string]any, len(v))
        for _, item := range v {
            s := fmt.Sprintf("%v", item)
            enumMap[s] = s
        }
        vm.scope.Declare(name, enumMap, "enum")
    case map[string]any:
        vm.scope.Declare(name, v, "enum")
    }
}
```

**Compile-time protection:** `set` targeting enum name = compile error (same as constants).

## 6. Migration Notes

### 6.1 Backward Compatibility

All changes are **additive**:
- New step types (`sleep`, `retry`, `assert`) — existing programs don't use them
- New top-level keys (`constants`, `enums`) — optional, ignored if absent
- Lambda in expressions — only activated by `fn(` prefix, no conflict with existing expressions
- Dynamic call_ref — fallback behavior, existing `call` works unchanged

### 6.2 Reserved Names

New reserved names that cannot be used as variable names:
- Enum names (e.g., `Status`, `Priority`) — if defined in `enums` block
- Constant names (e.g., `MAX_RETRIES`) — if defined in `constants` block

### 6.3 `mapFn` vs `map`

`map(arr, .field)` (expr-lang predicate) and `mapFn(arr, lambda)` (go-json higher-order) coexist:
- Use `map` for simple field extraction: `map(users, .name)`
- Use `mapFn` for complex transforms: `mapFn(users, fn(u) => upper(u.name) + ' (' + string(u.age) + ')')`

## 10. Known Limitations (Intentional)

These are **design decisions**, not bugs. Each has a rationale and workaround.

### 10.1 Anonymous Lambda Cannot Recurse

**Behavior:** An **anonymous** lambda (`fn(x) => ...`) cannot call itself. Only **named** lambdas (`fn name(x) => ...`) support recursion.

```jsonc
// ❌ Anonymous — cannot recurse (no name to self-reference)
{"let": "factorial", "expr": "fn(n) => n <= 1 ? 1 : n * factorial(n - 1)"}

// ✅ Named — CAN recurse (name injected into own env)
{"let": "factorial", "expr": "fn factorial(n) => n <= 1 ? 1 : n * factorial(n - 1)"}
```

**Rationale:** Anonymous lambdas have no name to reference themselves. The variable name (`factorial` in `let`) is not yet in scope when the lambda is being compiled (snapshot capture). Named lambdas explicitly declare their self-reference name, which is injected into the lambda's eval environment.

**Depth limit:** Named recursive lambdas enforce `vm.limits.MaxDepth` (default 1000). Exceeding produces graceful error: `lambda 'factorial': recursion depth limit (1000) exceeded`.

**For deeper recursion:** Set higher limit via runtime option `WithLimits({MaxDepth: 5000})` or CLI flag `--max-depth 5000`. Program JSON `"limits"` can only lower, not raise (security by design).

### 10.2 Lambda Type Safety is Runtime-Only

**Behavior:** No compile-time type checking on lambda parameters or return values.

```jsonc
// This compiles fine but fails at runtime if items contains non-numbers
{"let": "doubled", "expr": "mapFn(items, fn(x) => x * 2)"}
```

**Rationale:** go-json uses gradual typing. The expression engine (expr-lang) evaluates expressions at runtime. Since lambda bodies are expr-lang expressions, they inherit the same runtime-only type checking. Adding compile-time type inference for lambdas would require:
- A full type inference engine (significant complexity)
- Breaking the "structural validation at compile time, expression validation at runtime" principle

**Workaround:** Use `assert` for runtime validation:

```jsonc
{"assert": "all(items, {# != nil && type(#) in ['int', 'float']})", "message": "'items must be numeric'"},
{"let": "doubled", "expr": "mapFn(items, fn(x) => x * 2)"}
```

### 10.3 Lambda Cannot Mutate Outer Scope

**Behavior:** Lambdas capture a snapshot — mutations inside a lambda do NOT affect the outer scope.

```jsonc
{"let": "counter", "value": 0},
{"let": "inc", "expr": "fn() => counter + 1"},
// inc() returns 1 every time — it doesn't mutate counter
```

**Rationale:** This is the same snapshot capture design as 10.1. Lambdas are pure functions — they take input and produce output. Side effects are handled at the step level (`set`, `call`), not inside expressions.

**Workaround:** Use `applyEach` with step-level mutation, or use `reduceFn` to accumulate state:

```jsonc
// Accumulate with reduceFn
{"let": "total", "expr": "reduceFn([1, 2, 3, 4, 5], fn(acc, x) => acc + x, 0)"}
```

### 10.4 Test Runner `before`/`after` Not Yet Executed

**Behavior:** The `before` and `after` blocks are parsed and stored but not executed before/after each test case.

**Rationale:** The test runner currently creates a fresh wrapper program per test case. Executing `before`/`after` steps requires either:
- Injecting them into each wrapper (adds complexity to temp file generation)
- Running them in a shared runtime context (breaks test isolation)

**Workaround:** Include setup logic directly in the test program being tested, or use a shared function that each test case calls.

### 10.5 Test Runner Per-Case Timeout Not Enforced

**Behavior:** The `timeout` field on test cases is parsed but not enforced at runtime.

**Rationale:** The `Runtime.Execute()` API doesn't support per-call timeout options. Adding this requires an API change (`ExecuteOption` pattern). Program-level `limits.timeout` still works for overall execution timeout.

**Workaround:** Set `"limits": {"timeout": "5s"}` in the program being tested.

## 11. Future Considerations

### 11.1 Lambda-Based Higher-Order Additions (Post-4.5g)

With lambda support now available, the following functions become natural additions to `stdlib/higher_order.go`:

| Function | Signature | Description |
|----------|-----------|-------------|
| `rejectFn(arr, fn)` | Inverse of `filterFn` — keep items where fn returns false |
| `takeWhileFn(arr, fn)` | Take items from start while fn returns true |
| `dropWhileFn(arr, fn)` | Skip items from start while fn returns true |
| `findFn(arr, fn)` | Return first item where fn returns true |
| `everyFn(arr, fn)` | True if fn returns true for all items |
| `someFn(arr, fn)` | True if fn returns true for any item |
| `tap(value, fn)` | Call fn(value) for side-effect, return value unchanged |
| `identity(x)` | Return x unchanged (useful as default transformer) |

These do NOT require expr-lang predicate support — they use the same `func(...any)(any,error)` callable pattern as `mapFn`/`filterFn`.

**Note:** expr-lang already has `filter`, `map`, `all`, `any`, `find` with predicate syntax (`.field` or `{# > 0}`). The `*Fn` variants are for when you need a **callable** (lambda or named function) instead of a predicate expression. Both coexist:
- `filter(users, .active)` — expr-lang predicate (field access)
- `filterFn(users, fn(u) => u.age > 18 && u.active)` — lambda (complex logic)
